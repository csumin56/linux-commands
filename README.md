# 리눅스 프로세스 관리 명령어

리눅스에서 프로세스를 확인하고 제어할 때 사용하는 명령어 4가지를 정리합니다.

---

## 목차
- [top](#top)
- [ps](#ps)
- [jobs](#jobs)
- [kill](#kill)

---

## top

실행 중인 프로세스를 실시간으로 모니터링하는 명령어입니다.

```bash
$ top
```

### 주요 옵션

| 옵션 | 설명 |
|------|------|
| `top -d <초>` | 화면 갱신 주기 설정 (기본 3초) |
| `top -n <횟수>` | 지정한 횟수만큼 출력 후 종료 |
| `top -p <PID>` | 특정 PID만 모니터링 |
| `top -u <사용자>` | 특정 사용자의 프로세스만 표시 |

### 실행 중 단축키

| 키 | 동작 |
|----|------|
| `q` | 종료 |
| `k` | 프로세스 종료 |
| `M` | 메모리 사용률 기준 정렬 |
| `P` | CPU 사용률 기준 정렬 |

## ps

현재 실행 중인 프로세스의 목록을 출력하는 명령어입니다. `top`과 달리 실시간 갱신 없이 현재 상태의 스냅샷을 보여줍니다.

```bash
$ ps
$ ps aux          # 모든 프로세스 상세 출력
$ ps aux | grep firefox   # 특정 프로세스 검색
```

### 주요 옵션

| 옵션 | 설명 |
|------|------|
| `a` | 모든 사용자의 프로세스 표시 |
| `u` | CPU, 메모리 등 상세 정보 포함 |
| `x` | 터미널 없이 실행 중인 프로세스도 표시 |
| `f` | 계층 구조(트리) 형태로 표시 |

### ps aux 출력 항목

| 항목 | 설명 |
|------|------|
| PID | 프로세스 ID |
| %CPU | CPU 사용률 |
| %MEM | 메모리 사용률 |
| STAT | 프로세스 상태 (R: 실행, S: 대기, Z: 좀비) |
| COMMAND | 실행 명령 |

## jobs

현재 셸 세션에서 실행 중인 백그라운드 작업 목록을 보여주는 명령어입니다.

```bash
$ sleep 100 &    # 백그라운드 실행
[1] 3124

$ jobs
[1]+  Running    sleep 100 &
```

### 주요 옵션

| 옵션 | 설명 |
|------|------|
| `jobs -l` | PID 포함하여 표시 |
| `jobs -r` | 실행 중인 작업만 표시 |
| `jobs -s` | 정지된 작업만 표시 |

### 관련 명령어

- `fg %1` : 1번 작업을 포그라운드로 전환
- `bg %1` : 1번 작업을 백그라운드로 전환
- `Ctrl+Z` : 현재 작업 일시 정지

---

## kill

프로세스에 시그널을 보내는 명령어입니다. 주로 프로세스 종료에 사용합니다.

```bash
$ kill 1234         # 기본 종료 (SIGTERM)
$ kill -9 1234      # 강제 종료 (SIGKILL)
$ kill %1           # jobs의 1번 작업 종료
```

### 주요 시그널

| 번호 | 이름 | 설명 |
|------|------|------|
| 2 | SIGINT | 인터럽트 (Ctrl+C와 동일) |
| 9 | SIGKILL | 강제 종료 (무시 불가) |
| 15 | SIGTERM | 정상 종료 요청 (기본값) |

> `kill -9`는 프로세스가 응답하지 않을 때 사용합니다. 먼저 `kill`로 정상 종료를 시도하는 것이 좋습니다.
